Activación en Servidor

En esta sección se explicará cómo acceder al cluster desde una terminal. Así mismo se describirá cómo activar GAMOS en los servidores DZero y DZero1 para su posterior ejecución. Por último se explicarán algunos comandos de terminal útiles que podrían mejorar la experiencia de usuario con el uso de GAMOS.

Acceder a DZero o Dzero1

Para acceder a DZero basta con escribir:

ssh -Y user@dzero.uniandes.edu.co

Mientras que para acceder a DZero1 se necesita ejecutar el siguiente comando:

ssh -Y user@dzero1.uniandes.edu.co

En los anteriores comandos la palabra user debe cambiarse por el usuario de correo electrónico de la Universidad de los Andes. Posteriormente se les pedirá la contraseña personal y una vez la introduzcan ya estarán dentro del servidor escogido, ya sea Dzero o Dzero1.

Activación de GAMOS en DZero o DZero1

Una vez se ha entrado al cluster, el siguiente paso a seguir es activar GAMOS a través de la ejecución de un script de bash. Los scripts necesarios pueden ser descargados ejecutando el siguiente comando:

git clone https://github.com/Altas-Energias/scripts-GAMOS.git

Una vez descargados los scripts, debe moverse a la carpeta que los contiene. Para esto debe ejecutar:

cd scripts-GAMOS

Antes de ejecutar alguno de los scripts, primero debe cambiar la palabra user dentro de cada uno de ellos por el usuario de su correo electrónico institucional. Esto se puede lograr usando alguno de los editores de texto que tiene por defecto cada servidor. Algunos de los editores de texto que puede usar son gedit, nano o vim. Si por ejemplo decide modificar el script iniciar_gamos_dzero.sh con nano, debe ejecutar:

nano iniciar_gamos_dzero.sh

Cuando esté dentro del editor de texto, debe ir hasta donde se encuentra la palabra user usando las flechas del teclado y debe borrar la palabra, cambiandola por su usuario de correo institucional. Una vez haya puesto su usuario, puede salir del editor de texto presionando Ctrl+x. Antes de salir se le preguntará si desea guardar el archivo, a lo que usted debe responder Si o simplemente S. Posteriormente se le preguntará con qué nombre quiere guardar el archivo. Si desea mantener el nombre del archivo solo debe presionar Enter y si no puede cambiarlo y luego presionar Enter. Luego de modificar cada uno de los scripts puede ejecutarlos a través del comando source. Si desea activar GAMOS DZero puede ejecutar:

source iniciar_gamos_dzero.sh

o si desea activar DZero1 puede usar:

source iniciar_gamos_dzero1.sh

y ya GAMOS quedaría listo para ser usado. Los scripts por defecto, luego de ser ejecutados, cambian la ubicación del usuario, de manera que ya no estarán en la carpeta scripts-GAMOS sino que serán trasladados a la ubicación a la que por defecto un usuario entra cuando accede a alguno de los servidores. Esta ubicación por defecto es /home/user/, donde user es su usuario de correo institucional.

Comandos de consola útiles

Algunos comandos de consola útiles para navegar dentro de DZero o DZero1 y mejorar la experiencia de cualquier usuario son las siguientes:

  • ls: Permite visualizar todo lo que se encuentre dentro de la carpeta en la cual está actualmente
  • pwd: Permite obtener la dirección (o ruta) de la carpeta en la cual está actualmente.
  • cd destino: Permite moverse desde la carpeta actual hasta la carpeta de destino. El destino puede ser tanto una carpeta específica como una dirección. Para moverse a la carpeta que contiene a aquella en la cual está actualmente ubicado basta con ejecutar cd ..
  • mkdir nombre_de_carpeta: Permite crear una carpeta con nombre nombre_de_carpeta dentro de la carpeta en la cual se encuentra actualmente.
  • rm archivo_o_carpeta: Permite eliminar tanto archivos como directorios. Cuando hay que eliminar una carpeta (con todo su contenido dentro) se usa, luego de rm, la opción (o flag) -R.
  • mv elemento destino: El comando mv, en principio, sirve para mover un elemento (archivo o carpeta) desde el punto donde se encuentra hasta un destino. También el comando mv sirve para cambiar el nombre de un archivo o carpeta. En este caso en particular en el lugar de elemento se pone el nombre del archivo al cual queremos cambiar su nombre y en el destino se escribe el nuevo nombre.
  • mv elemento destino: El comando mv, en principio, sirve para mover un elemento (archivo o carpeta) desde el punto donde se encuentra hasta un destino. También el comando mv sirve para cambiar el nombre de un archivo o carpeta. En este caso en particular en el lugar de elemento se pone el nombre del archivo al cual queremos cambiar su nombre y en el destino se escribe el nuevo nombre.
  • cp elemento destino: Permite copiar un elemento y mandarlo a un destino específico. Si se requiere copiar una carpeta, es necesario que luego de cp se use la opción (flag) -r.
  • scp -r archivo_o_carpeta user@dzero.uniandes.edu.co:/home/user : Permite subir tanto un solo archivo como una carpeta desde el PC personal a la carpeta por defecto de Dzero (/home/user). Si se desea subir el archivo o carpeta a DZero1 basta con cambiar DZero por DZero1 en el comando ejecutado. En archivo_o_carpeta se ubica el nombre del archivo o carpeta que se desea subir al servidor. También puede ubicarse la dirección (o ruta) del archivo o carpeta. Cabe resaltar que este comando se ejecuta estando fuera del servidor.
  • scp -r user@dzero.uniandes.edu.co:/home/user/archivo_o_carpeta direccion : Permite descargar tanto un solo archivo como una carpeta desde el servidor a una carpeta con cierta direccion (o ruta) dentro de su computador personal. Si se desea descargar el archivo o carpeta desde DZero1 basta con cambiar DZero por DZero1 en el comando ejecutado. En archivo_o_carpeta se escribe el nombre del archivo o carpeta que se desea descargar desde el servidor (siempre y cuando este se encuentre dentro de /home/user/). Si desea descargar un archivo o carpeta desde otra ruta que no sea /home/user/ puede cambiar sin problemas /home/user/archivo_o_carpeta por la dirección que desee dentro del servidor. Cabe resaltar que este comando se ejecuta estando fuera del servidor.
